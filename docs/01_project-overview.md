# プロジェクト概要

## 1. 何を作るか

YouTubeのタイトル、サムネイル、コメント、関連動画から、ユーザーが避けたいネタバレを隠すChrome拡張機能を作る。

ユーザーは作品やイベント単位でグループを作り、複数のキーワードを登録する。拡張機能はYouTube上の表示内容をブラウザ内で判定し、該当箇所を伏字・差し替え・非表示にする。

専用Webサイト、サーバー、バックエンド、外部APIは使わない。設定と判定はすべてChrome拡張機能内で完結させる。

## 2. 誰のためのプロダクトか

主な対象は、情報をリアルタイムで受け取れないため、後から動画を視聴する人である。

- スポーツ中継を録画や見逃し配信で観る人
- 映画、ドラマ、アニメ、ゲーム実況を自分のペースで追う人
- 大会やイベントの結果を知らずにアーカイブを観たい人
- YouTubeのホーム画面やおすすめ欄で意図せず結果を知ってしまう人

## 3. 解決する課題

ネタバレは、観たい動画を開く前に発生する。YouTubeでは、タイトルだけでなくサムネイル、関連動画、コメントにも勝敗や展開が表示されるため、テキストだけを隠しても不十分である。

本プロダクトは、ユーザー自身が「今は知りたくない対象」を登録し、次の複数経路をまとめて遮断する。

- タイトル: 一致部分を伏字化する
- サムネイル: 内容が見えない画像へ差し替える
- コメント: ネタバレにつながる領域を非表示にする
- おすすめ・関連動画: ネタバレにつながる領域を非表示にする

## 4. MVPの利用の流れ

1. ユーザーが拡張機能のpopupを開く。
2. 「W杯2026」のようなグループを作る。
3. 「W杯2026」「ワールドカップ」「アルゼンチン」のようなキーワードを登録する。
4. popupが設定を `chrome.storage.local` に保存する。
5. 開いているYouTubeのcontent scriptが変更を検知する。
6. 現在表示されている要素と、その後に読み込まれた要素へルールを適用する。

設定変更後にYouTubeタブを再読み込みしなくても反映されることを目標とする。

## 5. 実装優先順位

### 最優先: ネタバレ防止の核

- キーワードグループの追加・削除・一覧表示
- 有効なキーワードとの一致判定
- タイトルの伏字化
- サムネイルの差し替え
- コメント・おすすめ欄の非表示
- 設定変更のリアルタイム反映
- YouTubeの後読み込み要素への追従

### 余力があれば

- 視聴済み管理: グループの `enabled` を `false` にする
- 動画終了イベントの検知
- キーワード提案UI

### 将来拡張

- YouTube以外のサイトへの対応
- 同義語や関連語を含む高度な表記ゆれ対応
- 視聴済み状態の自動更新
- キーワードリストの共有

## 6. 今回やらないこと

- 専用Webサイトの開発
- サーバー、バックエンド、データベースの構築
- 外部APIとの通信
- ユーザーアカウントやログイン
- AIによる文章・画像内容の判定
- 全サイト共通の汎用ブロッカー
- 全角半角以外の高度な表記ゆれ・同義語判定

スコープを広げる場合は、MVPの核が動くことを確認してから別タスクとして扱う。

## 7. 設計上の基本方針

- ユーザーの設定は端末内の `chrome.storage.local` にだけ保存する。
- popup/optionsとcontent scriptの責務を分ける。
- YouTube固有のDOMセレクターや書き換え規則を一箇所に集約し、将来ほかのサイト用ルールを追加できる形にする。
- YouTubeのDOM変更に備え、特定のセレクターが壊れても全機能が停止しないよう機能単位で処理を分ける。
- DOM書き換えは、同じ要素へ何度実行しても結果が壊れない「冪等」な処理にする。
- 権限は必要最小限にし、YouTube以外の閲覧内容へアクセスしない。

詳しい構成は [03_architecture.md](./03_architecture.md) を参照する。
